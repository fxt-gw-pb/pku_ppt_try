# PKU Red Defense PPT

A data-driven static HTML PPT template in the visual language of 北京大学 academic defense decks (PKU red `#9A0000`, white content pages, 16:9 / 1280×720, top-right chapter nav, bottom-right page numbers). Decks are generated by writing `data/slides.json` — never by hand-authoring HTML.

## Use When

- 论文答辩 / 毕业答辩 (thesis defense)
- 开题报告 / 中期检查 (opening / mid-term report)
- 学术汇报、组会、课题汇报 (research presentation, lab meeting)
- Any Chinese-academic-style HTML deck where a red-accented, restrained, paper-grade layout is appropriate.

**Do NOT use when** the user wants a marketing deck, product launch, landing-page style, glassmorphism, purple/blue tech gradient, dark hero pages, or a generic English business slide template. This is an academic deck — keep it academic.

## Workflow

1. **Understand the source.** Read the user's outline / manuscript / abstract. Identify: topic, presenter, advisor, school/department, date, and (most importantly) the list of section headings and the user-supplied images.
2. **Bucket into 3–6 chapters.** Default to the five-part defense structure when in doubt: 背景和意义 / 综述和评述 / 思路和内容 / 过程和方法 / 成果与展望. Runtime supports 3–6; the top-right nav distributes automatically.
3. **Pick a layout per slide.** Consult `references/layout-selection.md` for the content-type → layout mapping. Default deck shape: `cover` → `contents` → for each chapter: `section-divider` + 2–4 content layouts → `closing`.
4. **Classify each image.** Photo / chart / paper-screenshot / diagram / logo. Resolve to a `fit` value per `references/image-rules.md`. Never let users' images get stretched — pick `contain` or `diagram` for anything with text inside.
5. **Write `slides.json`.** Use the schema in `references/slides-json-schema.md`. Keep `meta`, `chapters`, and `slides[]` strictly to the documented fields. Use the rich-text vocab (`**…**`, `<span class="accent">…</span>`, `<em>…</em>`, `<br>`) sparingly — only on truly key phrases.
6. **Materialize the deck.** Either:
   - Run `python3 scripts/create_deck.py <slides.json> <output-dir>` to copy `assets/template/` to a new folder and drop the new JSON in, **or**
   - Copy `assets/template/` manually and overwrite `data/slides.json` and `assets/media/`.
7. **Validate.** Run `python3 scripts/validate_slides.py <output-dir>/data/slides.json` from the deck root. Catches: unknown layouts, missing required fields, broken image paths.
8. **Preview.** `cd <output-dir> && python3 -m http.server 8080`, open `http://localhost:8080/index.html`. Verify visually against `references/visual-reference/` thumbnails. PDF export = browser Print.

## Template assets

`assets/template/` is a complete, runnable deck:

```
assets/template/
├── index.html                  # Deck shell; do not edit
├── deck-stage.js               # Generic stage component (scale, nav, print, thumbnails)
├── data/slides.json            # ← THIS is what you generate per user
└── assets/
    ├── theme-pku-red.css       # Color + font tokens; edit only to retune the palette
    ├── base.css                # All chrome + layout CSS (single file)
    ├── runtime.js              # Reads slides.json, dispatches to layout fns
    └── media/                  # Sample images; replace with user images
```

The runtime accepts an inline data source for environments without a server: set `window.__PKU_DECK_INLINE = { meta, chapters, slides }` before `runtime.js` runs.

## Validation checklist

Before reporting the deck as done:

- [ ] `index.html` loads in a browser served via HTTP (not `file://`).
- [ ] The chapter nav in the top-right shows the correct 3–6 chapters.
- [ ] No page is missing its `chapterIndex` (chapter underline won't appear if missing).
- [ ] No image is visibly stretched. Charts/screenshots use `contain` or `diagram`.
- [ ] PKU red `#9A0000` is the only accent color. No purple/blue gradients, no glassmorphism, no marketing card stacks, no dark hero.
- [ ] `python3 scripts/validate_slides.py` returns 0.
- [ ] Page numbers on content pages run consecutively (chapter dividers correctly skipped).

## Style discipline

PKU red is reserved for: top chapter nav underline, ordinal numbers, key phrases (`<em>` / `<span class="accent">`), section-divider full-bleed backgrounds, cover background. Everything else is white pages, light-gray card borders, no heavy drop shadows. If a page would not look out of place in a serious 北大 thesis defense, you are on the right track.
